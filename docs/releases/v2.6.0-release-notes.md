# Nightshade 2.6.0 — Plan Tonight, NINA Import, Mobile Companion, Web Dashboard

**Release date:** _TBD (draft)_
**Branch:** `release/v2.5.0-hardening`
**Compared to:** v2.5.0 tag (2026-02-06, "Stub Cleanup & New Icon")
**Commit count:** 210 on the audit branch + a 914-file / +269,063 LOC squashed "pre-hardening" baseline from the 3-month dev window between Feb and May
**Full changelog:** [v2.5.0...v2.6.0](https://github.com/Scoduglas1999/Nightshade/compare/v2.5.0...v2.6.0)

**Companion document:** `CHANGELOG.md` (in the repo root) carries the engineering-grade detail with `§N.M` cross-references back to the audit doc

This is the largest Nightshade release to date. The published v2.5.0 (Feb 2026) was a tiny "icon + stub-cleanup" release that captured almost none of the work in progress at the time. Three months of development followed — building Plan Tonight, NINA / SGP import, the real web dashboard, the real mobile companion, the science backend, the defect-map pipeline, a complete pre-release audit-driven hardening pass, then 14 waves of code-quality work, then a feature-completeness audit + fix pass — and it all lands here as v2.6.0.

If you are upgrading from 2.5.0, your existing sequences, equipment profiles, and capture data all load forward. The only "breaking" change is an internal package rename (`nightshade_webrtc` → `nightshade_remote_protocol`) that only affects plugin authors; end users are unaffected.

## TL;DR — what's new at a glance

1. **Plan Tonight** — a real dynamic scheduler (RoboTarget-class) that picks targets across your night, weights by altitude/moon/weather/priority/scheduled-window, explains its decisions, tracks per-target multi-night progress, and re-evaluates on hardware events (not just on a wall-clock timer).
2. **NINA and Sequence Generator Pro sequence import** with auto-format detection, a node-mapping table preview, and "force-import" that preserves the raw scalar fields of unsupported nodes instead of silently dropping them.
3. **Real web dashboard** in any browser. Full equipment control: cooling, gain, offset, readout mode, binning, subframe, filter wheel, focuser, rotator, mount slew, sequence load + checkpoint resume + profile switch, analytics, plate-solve wizard, polar-align wizard, flat wizard, mosaic planner, framing assistant. Phone-tailored bottom-tab layout at ≤ 600 px.
4. **Real mobile companion** (iOS + Android). QR-code pairing, paired-devices table, phone-tailored bottom-tab dashboard, push notifications for critical events, foreground service with capture progress percent, deep-links from notifications.
5. **Defect-map / bad-pixel correction** so users can image without darks and still get clean lights.
6. **Plate-solver detection + setup UX** with auto-detect for ASTAP and astrometry.net across all platforms, catalog presence checks, and a verify-solve button that runs the binary at settings-save time.
7. **All-sky polar alignment** (Sharpcap-style single-shot) alongside the existing 3-point TPPA.
8. **Catalog overlay** on plate-solved frames (Messier / NGC / IC + bright stars with configurable magnitude limit).
9. **End-of-session report** + multi-night campaign roll-up per target.
10. **Disk-space guard** (pre-flight + watchdog + dashboard tile).
11. **First-night tutorial** + 10-step equipment onboarding wizard.
12. **The imaging pipeline actually works.** v2.5.0 had a fake plate solver returning commanded coordinates, a flat wizard taking flats at gain=0 regardless of camera profile, FITS double-scaling, FWHM overstated by 25%, and an OTA updater that silently corrupted installs on locked files. All of these are fixed.

## Brand-new major features

### Plan Tonight — dynamic scheduler (RoboTarget-class)

A target-selection engine for unattended multi-target nights. Routed at `/scheduler` and consumed as the **Scheduler tab inside Plan Tonight** in production (the legacy `/scheduler` route is still a redirect for external bookmarks).

- **Per-target integration goals by filter** — e.g. `L: 4h, R/G/B: 2h each, Hα: 6h` — and per-target constraints: time window (UTC or "after astronomical twilight"), moon illumination cap, minimum moon separation, custom horizon profile (samples-based, separate from the legacy 8-point compass mask so tree-lines and roof obstructions are honored).
- **Hysteresis-stable target selection.** A challenger only displaces the running target when its weighted score exceeds the current target's by the configured `hysteresisRatio` (default 1.20×). No more ping-pong between two equally-good targets.
- **Automatic re-evaluation on EventBus channels** for weather, guiding, and mount-state changes — not just on a wall-clock timer — so a guiding excursion or an incoming cloud bank can flip targets inside the next decision tick.
- **Decision panel surfaces the full scoring breakdown** (per-factor value, weight, weighted contribution) + the runner-up + a countdown to next eval + a force re-evaluate button.
- **20 new tests** cover engine selection / hysteresis / lifecycle, the Meeus JD references and NOAA sun-alt-az path in `sky_calculations`, and the two `SchedulerScreen` widget states.
- **Plan Tonight integration**: in v2.6 the Scheduler is consolidated as a tab inside Plan Tonight alongside a new Progress tab. Target progress + ETA service runs in the background.
- **Mosaic + scheduler composition** is correct in v2.6 — a multi-panel mosaic added to the scheduler captures every panel, not just the first (this was silently broken before the May 16 audit pass).

### NINA / SGP sequence import

Drop-in migration from the two competitor sequencers users are most likely to be coming from.

- **Where**: Sequencer screen → Import sequence (toolbar). File picker → auto-format detection → summary dialog showing node-mapping table → user persists.
- **Auto-format detection** by file extension and content sniffing (`.json` / `$type:NINA…` → NINA; `.sgf` / SGP JSON shape → SGP). Wrong extension still resolves to the right parser when the contents are unambiguous.
- **Node mapping table** in the summary dialog (source type → Nightshade type → count) so you see exactly what's about to be imported before committing.
- **Force-import on unsupported override**: the default is to raise a structured `UnsupportedNodeError` with the offending node type; toggling "import anyway" preserves the raw scalar fields of unsupported nodes and continues. No silent data loss.
- **Two persistence destinations**: "Open in editor" (load into active sequencer) and "Save to library" (persist via SequenceRepository without disturbing the current edit).
- **`docs/sequence-import-formats.md`** documents the NINA `$type` table, the SGP key table, drop / unsupported semantics, and two worked examples for support.

### Defect-map / bad-pixel correction pipeline

End-to-end cosmetic correction so users can image **without dark frames** while still hiding hot pixels, cold pixels, and small dust shadows.

- **Where**: Imaging screen → Image calibration section. Build, toggle "apply during capture", clear, and last-rebuild status all live here.
- **Builds a defect map from a small stack of dark frames** (minimum 5 by default; fewer is a structured error, not a silent fallback). Stored as a persistent `.ndm` file keyed by camera + sensor size + temperature bucket.
- **Multi-temperature buckets** so a −10 °C cooled run and a +5 °C cooled run don't share the same hot-pixel list. Lookup uses `temperatureBucketDecicelsius` (tenths of a degree) so a 1-degree rebuild interval is meaningful.
- **Neighborhood-median repair at capture time**: defective pixels are replaced by the median of their valid neighbors before the frame is written. Lights are clean as they leave the camera; downstream stacking sees no artifacts.
- Disabled-state UI: every control disables with an explanatory tooltip when the camera is disconnected, sensor dimensions are unknown, or no cooler telemetry has arrived yet.
- **12 new Rust unit tests** for the defect-detection statistics.

### Plate-solver detection & setup UX

A first-class plate-solver configuration surface designed to eliminate the "why doesn't plate solve work?" friction.

- **Where**: Settings → Plate Solving (`/settings/plate-solving`). Reusable `PlateSolverRequiredBanner` also appears in centering, framing, and polar-alignment flows when no solver is reachable, with a `go_router` CTA straight to the settings screen.
- **Auto-detect ASTAP and Astrometry.net** across every standard install path on Windows, macOS, Linux — including ASTAP CLI / GUI builds, Homebrew `astrometry.net`, and Cygwin `solve-field`. Catalog presence detected separately so "binary OK, no catalog" is a distinct state.
- **Verify-solve button** runs the configured binary to confirm it works on this machine — catches stale binaries, missing dependencies, wrong-architecture downloads at settings-save time rather than mid-sequence.
- **Auto-fallback chain**: when both solvers are installed and the user picks "Auto", a failure on one falls back to the other for that solve attempt only.
- The previous "ASTAP not found" snackbar in `CenteringDialog` has been replaced with the required-banner + settings CTA — the user is never stranded.
- **The internal "fake" solver** that returned commanded RA/Dec verbatim and labeled the result `success: true` has been **removed from the public solve path.** Plate solving now requires ASTAP or astrometry.net.

### Plate-solved frame catalog overlay

A new differentiator-class feature: once a frame is plate-solved, toggle a layer that overlays catalog objects on the preview.

- **Sources**: OpenNGC (DSOs) + HYG (stars), both reused from the existing planetarium catalog. No new downloads.
- **Real TAN gnomonic projection**, not a small-angle approximation. Forward/inverse roundtrip accurate to < 0.001 px.
- **FOV bounding box with pole-touching detection** for high-declination targets.
- **Magnitude filter** + DSO/star toggles + per-magnitude hit-radius + click-for-details panel.
- **Performance**: ≤ 800 ms for a synthetic 13k-DSO + 120k-star catalog over a 1 deg FOV (perf test gated in CI).
- **Configurable bbox padding** (default 0.05) for large nearby galaxies (Andromeda, LMC).
- **Auto-disposes on widget unmount** — no memory leak as the user pans/zooms across different WCS.

### End-of-session report

When a sequence completes (success, failure, or aborted), an automatic summary opens.

- **Per-target / per-filter rollup**: attempted / accepted / rejected frame counts, with rejection-reason histogram. Integration time per filter + total. Mean HFR, FWHM (computed as `2.0 × HFR`), star count, SNR proxy (background / noise).
- **Whole-session guide stats**: mean / max RA / Dec / total RMS + unguided-frame fraction.
- **Mount stats**: autofocus run count, meridian flip count, dither count, trigger fires.
- **Errors and warnings** emitted during the run.
- **Copy as Markdown** (deterministic, shareable) + **Export to .txt**.
- Auto-opens on the running → idle/completed/failed transition. Also reachable from the analytics screen for past sessions.

### Multi-night campaign roll-up per target

Aggregated view in the analytics tab.

- Session count + date range.
- Per-filter captured frames + integration time + integration-goal progress (frames + integration hours).
- Mean HFR (frame-count weighted), mean seeing, mean effective imaging fraction across closed sessions only.
- Case-insensitive filter grouping (matches scheduler stack).
- Goal-only filters appear with 0% so brand-new campaigns show their planned filters rather than missing rows.
- Each session row deep-links into the per-session report dialog.

### All-sky polar alignment (Sharpcap-style)

A single-shot plate-solve mode added alongside the existing 3-point TPPA flow.

- Requires a plate solver (ASTAP, astrometry.net, or PlateSolve2).
- Iteration cadence (`iterationCadenceSecs`, default 3.0 s, floor 0.5 s) and acceptance threshold (`autoCompleteThreshold`, default 30″) both configurable.
- Auto-complete on 3 consecutive iterations below threshold (`AUTO_COMPLETE_HOLD_SECS`).
- Northern / Southern hemisphere flag captured; `autoCompleteThreshold` is now correctly forwarded for both TPPA and all-sky paths.

### Disk-space guard

- **Pre-flight check**: before a sequence starts, compute projected capture size (`width × height × bit_depth × frame_count × channels`) and compare to free space. Three states:
  - Informational: `M < N × 0.6`
  - Warning: `M > N × 0.6` (advisory, dismissible)
  - Blocking: `M > N − 2 GB safety margin` (override possible, but loud)
- **Mid-sequence watchdog**: default 30 s poll. Default warning threshold 10 GB free; default abort threshold 2 GB free. Both user-configurable. Hysteresis (must climb to 110% of warning / 200% of abort before re-arming) so a one-time blip doesn't spam alerts.
- **Storage tile on the dashboard** with stacked bar + threshold marks + projected-run usage indicator.
- Watchdog auto-pauses the run on blocking events; on resume it re-checks before reanimating.

## Authentication & security (entire subsystem, new in 2.6)

The headless API authentication and pairing surface was previously a thin bearer-token model with significant gaps. v2.6 ships a real auth subsystem.

### Pairing

- **`POST /api/pairing/start` and `POST /api/pairing/verify`** are now part of `HeadlessApiServer`. First-run web onboarding no longer requires reading the bearer token from the console.
- **`info.pairingSupported: true`** in the `/api/info` response.
- **Pairing-attempt LRU eviction map** so the rate-limit data structure cannot grow unbounded under attack.
- **Constant-time bearer-token comparison** shared across the headless API and the LAN-push receiver. Timing-side-channel extraction of tokens character-by-character on a slow link is mitigated. Token-bucket rate-limit on comparison failures prevents DoS via the comparison loop cost.
- **Mobile QR-pairing payload schema validation**, RFC1918 / link-local / `.local` host check, fingerprint-confirmation sheet before persisting any pairing.
- **Mobile token migration to `flutter_secure_storage`** (Keychain on iOS, EncryptedSharedPreferences on Android). First launch migrates existing tokens out of plaintext SharedPreferences.

### Browser auth

- **HttpOnly + `SameSite=Strict` cookie auth path** with per-session CSRF token for the dashboard "remember me" flow. Replaces the previous `localStorage`-backed bearer token storage.
- **Explicit CORS allow-list.** `Access-Control-Allow-Origin` is no longer reflected from the request `Origin` header on high-risk POSTs. A malicious local process on `127.0.0.1` can no longer script high-risk POSTs.
- **Dashboard CSP `connect-src` restricted to `'self' ws: wss:`.** Previous `http://*:* https://*:* ws://*:* wss://*:*` allow-anything policy is gone.
- **Dashboard startup warning** when bound on a non-loopback address: "Nightshade API listening on ALL interfaces. Anyone on your network can attempt to authenticate. Press Ctrl+C now if this is wrong."

### WebSocket auth

- **WebSocket auth via short-lived one-time tickets** issued by `POST /api/ws/ticket`. Eliminates the previous `?token=…` query-string WebSocket auth that ended up in HTTP access logs.
- The legacy `?token=` form is still accepted with a deprecation warning; **will be removed in v2.7.**

### Authorization

- **`/api/files/browse` path allow-list** (Documents, configured save path, configured backup dir, configured calibration-library dir). Paths that do not canonicalize under one of those roots are rejected.
- **`infile_select` / `dst_path` canonicalization** across the headless API path-accepting endpoints, not only `/api/files/browse`.
- **Token redaction in structured logs.** `main_headless.dart` startup banner no longer leaks the auto-generated bearer.
- **Sanitized error response bodies** — `e.toString()` is gone. Internal errors return `{"error":"internal_error","requestId":"<uuid>"}`; full exception goes only to the structured log.

### LAN push (OTA)

- **LAN-push manifest signature verified before any byte is written.**
- **1 GiB pre-signature size cap** to prevent zip-bomb / memory-exhaust attacks.
- **LAN-push receiver refuses to start** if `NIGHTSHADE_UPDATE_PUBLIC_KEY` was not compiled in — clear error rather than silent-disabled state.
- **Updater file lock** at `<install_dir>/updates/.updater.lock` prevents concurrent updater runs racing on `pending_install.json`.

## Imaging pipeline upgrades

### Stretch

- **MAD-based STF auto-stretch** matching the PixInsight reference: median ± `2.8 × 1.4826 × MAD`. Replaces the previous 0.001 / 0.999 percentile clip that left faint nebula and LRGB integrations pegged to bright stars.

### Plate solving (when an external solver is configured)

- **WCS keyword parse errors propagate.** `CRVAL1` / `CRVAL2` / `CD1_1` / `CD1_2` / `CD2_1` / `CD2_2` failures are no longer `unwrap_or(0.0)`'d. Malformed values surface as `PlateSolveError::WcsParse`.
- **Plate-solve `detect_local_maxima` threshold floor** now uses `N_sigma × background_sigma_estimate` from the existing background estimator instead of a fixed 250-ADU floor that was meaningless on 10-bit cameras and pre-stretched images.
- **Auto-fallback chain** between ASTAP and astrometry.net when "Auto" is selected.

### Alpaca

- **`ImageBytes` binary protocol support** with JSON fallback. The binary path sends `Accept: application/imagebytes` and parses the v3 binary header. **~10× throughput improvement over JSON for large frames.**
- **Rank-aware parsing** so color-sensor frames (rank 3) no longer have a channel dimension silently dropped. Pixel JSON values that fail to parse surface as `AlpacaError::ParseError { offset, found }` instead of silently substituting `0`.
- **Pooled HTTP client** with per-request timeouts via `RequestBuilder::timeout(...)`. Was rebuilding a fresh `reqwest::Client` per image.

### FITS

- **BSCALE / BZERO round-trip fixed.** Reading no longer leaves stale source BSCALE / BZERO entries in `header.keywords` after applying them to the data buffer. Writing emits only the freshly-computed values for the chosen output BITPIX.
- **COMMENT and HISTORY** cards now emitted as proper standard cards instead of malformed `COMMENT_<n> = ...` value cards.
- **String values padded to ≥ 8 chars** per FITS 4.2.1.1.
- **`NAXIS > 3` rejected explicitly** with `FitsError::Unsupported4DCube` instead of silently dropping planes.
- **XBAYROFF / YBAYROFF support** for correct subframe Bayer pattern composition when crop offsets are odd.

### XISF

- **`Creator` / `CreationTime` use the real package version** instead of hardcoded `"Nightshade 2.0"`.
- **Boolean parse accepts `True` / `TRUE`** per spec (was only accepting lowercase).
- **Real XISF version field** instead of hardcoded.

### Debayer

- **VNG threshold corrected** from `min_g + (max_g − min_g) × 3 / 2` (always greater than max — no edge preservation) to the intended `(min_g × 3) / 2`. Every OSC frame now keeps edge detail instead of silently degenerating to 8-direction averaging.

### HFR / FWHM

- **HFR median is a real median** — sort, filter NaN and ≤ 0, take the middle value. The previous `((a+b)/2 + c)/2 + d)/2` shape was an exponentially-weighted moving average labeled "median"; HFR-degraded triggers fired on single bad frames.
- **FWHM no longer overstated by 25%.** `calculate_hfr_at_point` now computes the true encircled-energy 50% half-flux radius matching NINA / SGP / PixInsight convention, and FWHM uses `2.0 × HFR`. Synthetic-Gaussian unit test gates the math.

### Airmass

- **Pickering airmass at altitudes below 10°** no longer silently uses an out-of-validity-range formula. Altitudes below 0° return `Err` (object below horizon); the wider altitude range uses **Young's 1994 formula**.

### Image processing pipeline

- **`process_tiled`** async vs sync paths reconciled.
- **`buffer_pool`** has bounded eviction.
- **`UNIFIED_IMAGE_STORAGE`** is now LRU-bounded with eviction events.
- **`ALPACA_CLIENTS`** lifecycle audited.
- **Thread-unsafe `simple_random`** replaced.
- **Per-tile normalization** documented and fixed.

### RAW (LibRaw)

- **`CString::new` failures** for bad-pixels / dark-frame paths return `RawError::InvalidPath { which, source }` instead of being silently ignored.
- **Stale "located by sRGB gamma defaults" comment** removed; code uses `nightshade_libraw_apply_config` correctly.

### PHD2 image format

- **Unknown `get_app_state` values** now map to `Phd2State::Unknown(String)` with a warning log instead of silently being treated as `Connected`.
- **`GuidingPaused` and `StarFound`** have explicit variants.

### Flat wizard (significantly rebuilt)

- **Captures with the camera's real gain and offset** read from the active equipment profile. **Flats taken at gain=100 will calibrate light frames captured at gain=100.** (Before v2.6, flats were hardcoded to `gain: 0, offset: 0` regardless of what your sequence used — flats wouldn't calibrate correctly.)
- **Rate tracking with sky-brightness compensation** for sky flats (twilight sequences).
- **Dark-flat capture support** via `FrameType.darkFlat`.
- **Subscribes to `cameraImageReady`** instead of `Future.delayed(exposureTime + 500ms)`. Slow USB / large-sensor captures no longer false-error.
- **Filter-level failures surface to the UI** instead of just `developer.log`.
- **Quick + Batch + Sky Flats** tabs.
- Cover/calibrator integration where available.

## Sequencer

### New recovery + trigger primitives

- **`RecoveryAction::Dither(DitherConfig)`** and **`RecoveryAction::Recenter`** variants. Standard "Dither Interval" and "Drift Limit" triggers now use them.
- **`TriggerType::DriftLimit { max_pixels: f64 }`** plate-solve drift trigger. Default action is `Recenter`.

### Database

- **Auto-save service**: configurable sequence-auto-save + backup interval + max-backups.

### OTA updater (substantially rewritten)

- **Locked files now retry up to 3 times**, then either schedule a delayed-until-reboot replace or fail the update with an actionable message ("Update could not replace foo.dll because it is in use. Please close Nightshade and try again."). Previous same-size optimization could silently corrupt installs with different-content-same-size files.
- **Backup is move-then-copy** — each destination file is renamed to `*.old` before the new file is written. On any failure all `*.old` files are renamed back; on success they're deleted. Covers every file the apply step touches.
- **Hash verification of every file** in the staging manifest after apply. Boot-time `verifyPendingInstall` re-hashes the executable against the recorded post-install hash and refuses to mark the update verified on mismatch.
- **File lock** at `<install_dir>/updates/.updater.lock` prevents concurrent updater runs.
- **`cleanup_staging` deletes `staging_dir`** itself, not its parent (the previous behavior wiped `backup/`, `pending_install.json`, and any future-staged update in one shot).
- **`staged_verified.marker`** written after manifest + package SHA-256 verification. `_bootstrapUpdater` refuses to copy `updater.exe` unless the marker matches the current manifest.
- **LAN-push package size cap** (1 GiB, configurable) enforced before manifest signature verification can write any byte.
- **LAN-push receiver refuses to start** if no `NIGHTSHADE_UPDATE_PUBLIC_KEY` was compiled in.
- **ZIP archives stream-extracted** via `archive`'s `InputFileStream` instead of being loaded whole into memory.
- **`Process.start` → `exit(0)` race** on updater launch hardened: app verifies `pid != 0`, sleeps briefly, flushes the logger, and on `Process.start` exception restores the staged-update status.

### Error propagation

- **Structured FFI error pipeline restored.** `NightshadeException._parseJson` actually `jsonDecode`s the Rust `ErrorInfo` payload instead of unconditionally returning `null`. Every typed `NightshadeError` variant survives across the boundary with `device_id`, `error_code`, `is_recoverable`, and `should_reconnect` intact.

## Compatibility notes

### For end users

- **No breaking changes.** Equipment profiles, sequences, capture data all upgrade in place.
- The audit-driven hardening fixed several silent-failure paths. If a feature suddenly "starts failing with an error" after upgrade, the failure was almost certainly happening before — you just couldn't see it. (Notable examples: flat wizard returning bogus exposures because of gain=0, plate solving "succeeding" by returning commanded coordinates, FITS frames with double-scaled BSCALE/BZERO.)

### For plugin / integration authors

- **`nightshade_webrtc` is now `nightshade_remote_protocol`.** Update your imports:
  ```dart
  // Before
  import 'package:nightshade_webrtc/nightshade_webrtc.dart';
  // After
  import 'package:nightshade_remote_protocol/nightshade_remote_protocol.dart';
  ```
- **WebRTC peer-connection / signaling primitives** were deleted. Remote control runs over REST + WebSocket via the headless API server. If you were importing `peer_connection.dart`, `signaling.dart`, or `secure_signaling_server.dart`, they are gone; use the headless API instead.
- **Renamed types**: `PlateSolveResult`, `MosaicConfig/Panel`, `CameraState`, `FocusDataPoint` — see Foundation Work → Type Renames above. Old names are gone, no shims.
- **`AppSettings`** dropped several decorative fields. Reads against them would have been reading nothing meaningful; consult the dedicated provider for that domain (`calibrationSettingsProvider`, `weatherSettingsProvider`, `plateSolverPreferenceProvider`, etc.).
- **WebSocket auth via `?token=<bearer>`** is deprecated. Use `POST /api/ws/ticket` → `?ticket=<short-lived>`. Legacy `?token=` form still accepted with a deprecation warning in v2.6; will be removed in v2.7.

### Headless API

- **API version is `2.5.0`** in the `/api/info` response (the API surface remained stable across the v2.5→v2.6 app rename). Minimum supported client version is `2.4.0`. Clients older than `2.4.0` receive a clear "client too old" error.
- **All endpoints requiring fields now return structured `4xx`** with the missing-field name. Clients that previously assumed `500` for any failure should refresh.

## Known limitations

- **Mosaic + scheduler composition** captures one panel per scheduler tick. Multi-filter LRGB cycles within a single dispatch are not yet generated; a single dispatch picks the filter with the largest remaining count.
- **The internal "matcher" plate solver** is disabled in favor of ASTAP / astrometry.net / PlateSolve2. The UI banner guides you to install one. A real geometric matcher (4-star quad hashing or Tycho-2 / Gaia-DR3 subset) is on the v2.7 roadmap.
- **PlateSolve2 path** is wired but not validated against a real install in this release; ASTAP and astrometry.net are the recommended paths.
- **PlateSolveScreen Verify-Solve** runs `astap --help` / `solve-field --help` as a binary-health smoke check, not a real plate-solve against a synthetic field. A true end-to-end verify path is queued for v2.7.
- **DSLR / mirrorless support (libgphoto2)** is detected but not bundled. Install `libgphoto2` separately if you need DSLR camera support.
- **iOS background imaging**: the iOS companion intentionally surfaces a "device must remain in foreground" banner. Long-form `UIBackgroundModes` + BGProcessingTask deferred to v2.7 pending App Store review trade-offs.
- **Tonight Overview card** in the planetarium: providers exist; the UI surface defers to v2.7. The data backbone via `target_suggestion_provider` + `SessionOptimizerService` already ships as the recommendation card in the planner.
- **Linux / macOS builds**: tested via CI matrix, not exhaustively on hardware. Windows is the primary release target.
- **Vendor SDK migration to `VendorSdk` trait**: ZWO migrated as proof-of-pattern; 12 vendor migrations queued for v2.7.
- **SkyWatcher pier-side encoder decode**: returns `NotSupported` in v2.6 (better than fabricated values); encoder-based decode queued for v2.7.
- **NINA / SGP import test fixtures** were authored from public Advanced-Sequencer documentation. A verification pass against verbatim NINA exports is recommended before marketing "imports NINA sequences losslessly".
- **Sequencer `SafetyFailMode` dropdown UI** — all 3 modes (`failClosed` / `failOpen` / `warnOnly`) have real behavior in the provider, but the visible dropdown currently shows only `failClosed` as a guarded default. Exposing the 3-option choice will follow broader field testing.
- **Scheduler `userPriority` weight tuning** — the default weight may need adjustment so user priority alone can flip the hysteresis without help from other factors. Default weights ship as a workable starting point; tuning lands in a v2.6.x patch.
- **`DefectMapService` web target** — `PlatformInt64 lastRebuiltUnixSeconds` is assigned straight into a Dart `int`, which is safe on desktop / mobile (64-bit) but would not compile under web (`int` is JS number, `PlatformInt64` becomes `BigInt`). Web is not a supported target in v2.6.
- **Experimental built-in guider** — `native/.../bridge/src/builtin_guider.rs` + `models/backend/builtin_guider_config.dart` ship as a behind-the-scenes WIP for a future native (non-PHD2) guider. No UI surface; off by default. Subject to change without notice.
