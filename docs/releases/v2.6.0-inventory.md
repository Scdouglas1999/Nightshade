## B. Imaging pipeline upgrades

### Stretch
- **MAD-based STF auto-stretch** matching PixInsight: `median ± 2.8 × 1.4826 × MAD`.
  Replaces the 0.001 / 0.999 percentile clip. [CHANGELOG §"Changed" — auto-stretch],
  [commit `80db05c`], [audit §6.9].

### Plate solving (when an external solver is configured)
- **WCS keyword parse errors propagate.** `CRVAL1`/`CRVAL2`/`CD1_1`/`CD1_2`/`CD2_1`/
  `CD2_2` are no longer `unwrap_or(0.0)`'d. Malformed values surface as
  `PlateSolveError::WcsParse`. [audit §6.4], [commit `0f2ff33`].
- **`detect_local_maxima` threshold floor** now `N_sigma × background_sigma_estimate`
  from the existing background estimator instead of a fixed 250-ADU floor.
  [audit §6.15], [commit `568bfa4`].
- **Auto-fallback chain** between ASTAP / astrometry.net when "Auto" is selected.

### FITS
- **BSCALE/BZERO round-trip fixed**: read no longer leaves stale entries in
  `header.keywords`; write emits only the freshly-computed values for output
  BITPIX. [audit §6.3].
- **COMMENT and HISTORY** as proper standard cards instead of malformed
  `COMMENT_<n> = ...` value cards. [audit §6.5].
- **String values padded to ≥ 8 chars** per FITS 4.2.1.1.
- **`NAXIS > 3` rejected** with `FitsError::Unsupported4DCube` instead of
  silently dropping planes.
- **XBAYROFF / YBAYROFF** support for correct subframe Bayer pattern composition
  when crop offsets are odd. [audit §6.6].
- Files: `imaging/src/fits*.rs`. [commit `a9ec93c`].

### XISF
- **`Creator` / `CreationTime`** use the real package version instead of
  hardcoded `"Nightshade 2.0"`. [audit §6.22].
- **Boolean parse accepts `True` / `TRUE`** per spec. [audit §6.21].

### Debayer
- **VNG threshold corrected** from `min_g + (max_g − min_g) × 3 / 2` (always
  greater than max → no edge preservation) to `(min_g × 3) / 2`. [audit §6.12],
  [commit `13683e9`].

### HFR / FWHM
- **HFR median is a real sorted-array median**, NaN- and zero-filtered.
  Previous `((a+b)/2 + c)/2 + d)/2` shape was an EMA mislabelled "median";
  HFR-degraded triggers fired on single bad frames. [audit §1.2], [commit `57afcb9`].
- **FWHM no longer 25% overstated.** `calculate_hfr_at_point` now computes the
  true encircled-energy 50% half-flux radius (matches NINA / SGP / PixInsight);
  FWHM uses `2.0 × HFR`. [audit §6.11].

### Airmass
- **Pickering at < 10°** no longer silently uses out-of-validity formula.
  Altitudes < 0° return `Err` (below horizon); wider range uses Young's 1994.
  [audit §6.14], [commit `a9ec93c`].

### Image processing pipeline
- **`UNIFIED_IMAGE_STORAGE` LRU-bounded** with eviction events (was unbounded).
  [audit-rust §3.5], [commit `439b4af`].
- **`ALPACA_CLIENTS` lifecycle audited** for eviction on disconnect.
- **`process_tiled` async vs sync paths reconciled**; **`buffer_pool` bounded
  eviction**; thread-unsafe `simple_random` replaced; per-tile normalization
  documented and fixed.
- **`Arc<RwLock<RuntimeConfig>>`** shared by spawned executor task and
  instruction nodes for dither / location / filter-offset live updates.

### RAW (LibRaw)
- **`CString::new`** failures for bad-pixels / dark-frame paths return
  `RawError::InvalidPath { which, source }` instead of silent ignore. [audit §6.25].
- Stale "located by sRGB gamma defaults" comment removed; code uses
  `nightshade_libraw_apply_config` correctly. [audit §6.24].
- New: `imaging/src/libraw_shim.c`, `imaging/vendor/libraw/libraw_{const,types,version}.h`.

### PHD2
- **Unknown `get_app_state`** maps to `Phd2State::Unknown(String)` with warning,
  not silent `Connected`. `GuidingPaused` / `StarFound` explicit variants. [audit §6.23].
- **11 new JSON-RPC methods** (clearCalibration, flipCalibration, setLockPosition,
  getLockPosition, deselectStar, setAlgoParam, getAlgoParam, getAlgoParamNames,
  findStar, getStarImage + parameter-format fixes for `set_lock_position`,
  `set_paused`, `set_exposure`, `clear_calibration`). [release notes §"PHD2 integration"].
- **Auto-launch on connect** when `phd2Path` setting is set (`Process.start` +
  10 s startup grace). [audit-handoff §2.1 PHD2 row].
- **`Phd2Controller._init` event listener** guards every mutation with `mounted`.
  [audit-handoff §1.10].

### Flat wizard (substantially rebuilt)
- **Uses real gain/offset from active profile** instead of hardcoded `0, 0`.
  Flats taken at gain=100 now actually calibrate light frames at gain=100.
  [audit-handoff §1.1], [commit `2ebeff9`].
- **Rate tracking with sky-brightness compensation** for sky/twilight flats.
- **Dark-flat capture** via `FrameType.darkFlat`.
- **Subscribes to `cameraImageReady`** instead of `Future.delayed(exposureTime
  + 500ms)`. [audit §4.4].
- **Filter-level failures surface to UI** instead of just `developer.log`.
- Quick / Batch / Sky Flats tabs; cover/calibrator integration.
