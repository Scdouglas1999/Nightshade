> This document is the single source of truth for the cleanup work that follows the v2.5.0 hardening branch. Every recommendation is traced back to an audit-section ID in §5 (coverage matrix). Maintainers should read §1 for the verdict and §3 for the ship plan; agents pick up scope from §4.

## §1. Executive summary

The five audits converge on a unanimous verdict: **v2.5.0 is shippable as a hardening release, but ~2,250 LOC of confirmed dead code, four ship-blocker correctness bugs, and ~150,000 LOC of monolithic hand-written code must be addressed before v2.6 feature work resumes**. The architecture is sound (clean core ↦ app layering, structured `NightshadeError`, working fail-closed gate, correct WS-ticket auth model, comprehensive sequencer checkpoint). The debt is shape-of-code debt: a few enormous files (`api.rs` 9,770 LOC, `devices.rs` 8,854 LOC, `imaging_screen.dart` 7,079 LOC), a parallel deprecated-vocabulary problem (`DriverBackend` / `AvailableDevice` typedefs with 144 callers), 288 silent Rust fallbacks, and a structured-error pipeline that is a placebo because one Dart helper returns `null` unconditionally.

## §2. Cross-cutting themes

Each theme clusters findings from multiple audits.

### T1. Silent fallbacks (CRITICAL — direct CLAUDE.md violation)

**Problem.** CQ-RUST §4.3 finds **288 `unwrap_or_default()` / `unwrap_or(false)` / `unwrap_or(0)` sites with zero `// Why:` justifications**. CQ-OBSERVE §1c enumerates ~15 Dart `catch (_) → null/false/[]` swallows. CQ-RUST §1.5 finds 4 `if let Ok(_) = ...` discards. The Alpaca client alone has 8 `response.text().await.unwrap_or_default()` that mask HTTP failures (`alpaca/src/client.rs:315,521,580,631,668,722,776,820`). `alpaca/src/telescope.rs:957 is_pulse_guiding.unwrap_or(false)` silently reports "not guiding" on parse errors.

**File hot spots.** `bridge/src/device_capabilities.rs` (69), `bridge/src/devices.rs` (38), `bridge/src/api.rs` (31), `imaging/src/phd2.rs` (20), `enhanced_discovery.dart`, `plate_solver_utils.dart`, `coordinate_parser.dart`.

**Sizing.** **L** in aggregate; per-site is **S–M**. Each site needs a triage decision: convert to `?`, or add a `// Why:` comment with the invariant.

### T6. Disposal hygiene (CRITICAL — silent Timer/Stream leaks)

**Problem.** Three audits found the same bug class. CQ-DART §1f and §2d, CQ-RUST §2.1.

- `SequenceExecutionNotifier` (`providers/sequence_provider.dart:1549–1553`) holds `_progressTimer`, `_nativeEventSubscription`, `_checkpointTimer` — **no `@override dispose()`** (CQ-DART §1f).
- 9 device notifiers in `equipment_provider.dart:384, 550, 705, 825, 933, 1048, 1168, 1296, 1408` subscribe to `backend.eventStream` without storing a `StreamSubscription` (CQ-DART §1f).
- `framing_provider.dart:1515` debounce timer — no `dispose()` (CQ-DART §1f).
- 25 `Future.delayed` + `setState` sites without `mounted` guard or cancellable Timer (CQ-DART §2d). Worst: `flat_preview_panel.dart:566`, `meridian_flip_progress_dialog.dart:122`, `sequence_tree.dart:591`, `contextual_tour_prompt.dart:111`.
- 3 `setState`-after-`await` sites in `equipment_profiles_screen.dart:66,141,313` without `mounted` (CQ-DART §2e).
- 6 detached `tokio::spawn` with no panic supervision (CQ-RUST §2.1): `sequencer/executor.rs:658`, `bridge/api.rs:6489`, `bridge/api.rs:7755`, `imaging_ops.rs:61, 627`, `bridge/devices.rs:7186`.

**Sizing.** **M–L**. Reference impl exists: `weather_safety_provider.dart:128–130, 364–369`.

### T7. Error-UX inconsistency (HIGH)

**Problem.** CQ-OBSERVE §1, §3, §6.

- **Critical:** `NightshadeException._parseJson` is hard-coded `return null` (`nightshade_exception.dart:98-106`). Every Rust-side `to_json()` payload falls to string heuristics; `device_id`, `error_code`, `is_recoverable`, `should_reconnect` are all lost. **One-line fix; HIGH impact** (CQ-OBSERVE §1a).
- 13 fail-closed gate violations across `handlers_e_tostring` and `handlers_status_failed_string` rules in 5 handler files (CQ-OBSERVE §6a). The gate is operational but these slipped in.
- Mixed surfaces: SnackBar / dialog / inline banner / silent. Catalog-load and coordinate-parse paths silently show empty lists.
- 0 of 35 `NightshadeError` Display strings localized (LOW; defer).
- Bearer token logged plaintext at `main_headless.dart:236` (MED).

**Sizing.** **S** for `_parseJson`. **S** for the 13 fail-closed sites (gate finds them; helper exists). **M** for centralized error-surfacing pattern.

### T8. Test coverage gaps (HIGH)

**Problem.** CQ-TESTS §1 reports a **546:1 src:test ratio in `nightshade_app`** and **infinite** ratios in `nightshade_ui`, `nightshade_webrtc`, `apps/desktop`. The 10 largest untested files include `imaging_screen.dart` (7,079 LOC), `headless_api_server.dart` (1,352 LOC + 14 handler files), `ascom/windows_impl.rs` (3,756 LOC, **zero tests anywhere**).

**Required infrastructure (CQ-TESTS §6).**

1. `FakeNativeBridge` in `packages/nightshade_bridge/test/fakes/` → unlocks `FfiBackend` testing.
2. Widget-test harness `pumpAppScreen()` in `packages/nightshade_app/test/harness/` with `ProviderScope.overrides`.
3. `FakeAscomDriver` via `mockall::automock` on the COM trait boundary.
4. `FakeNightshadeNetworkClient` for `NetworkBackend` (3,010 LOC).

**CI gaps (CQ-TESTS §7).**

- No Windows-host job in `test-rust` matrix → `ascom/` never compile-tested.
- No `melos run audit:placeholders --fail-on-new-highrisk` gate.
- No pre-commit hooks (`.git/hooks/` empty).
- Coverage uses `fail_ci_if_error: false`; no threshold gate.

**Sizing.** **L**. The infrastructure is upfront; per-screen widget tests are M each after.

### T9. Observability bedrock (HIGH)

**Problem.** CQ-OBSERVE §4, §8, §9.

- **No `docs/RUNBOOK.md`** for 5 incident scenarios (frozen startup, plate-solve fails, OTA rollback, sequence resume, headless unreachable).
- No diagnostic dump screen; logs are exportable but not bundled with profile/sequence/system info.
- No SQLite corruption recovery (`MigrationStrategy.onCreate` throw kills first-launch on a corrupt `app_settings.db`).
- No telemetry — defensible for local-first ethos; opt-in crash uploader is the minimum-viable addition.

**Sizing.** **M–L**. Runbook is pure docs; SQLite recovery is M; crash uploader is L (and out-of-scope for v2.5.x — see §7).
